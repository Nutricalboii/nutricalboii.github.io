# Copy Fix Brief: De-AI'ing Text and Phrases

This guide highlights the shift from mechanical AI-generated sentence structures to organic, natural, human phrasing.

## Standing Style Rule

> [!IMPORTANT]
> Always avoid the typical AI antithesis pattern:
> - *"Not just X, but Y."*
> - *"That's not X. That's Y."*
> - *"It isn't X — it's Y."*
> - *"Which is precisely why..."*
>
> Replace these with straightforward, declarative statements of fact.

## Phrasing Updates

| Before (AI tic) | After (Natural/Organic) |
| :--- | :--- |
| **systemctl list-units:** "What's actually running. Not a highlight reel." | "Active processes and system status." |
| **Case Studies Subtitle:** "Not cards. Not badges. What the problem was..." | "A record of key challenges, architectural decisions, and post-mortems." |
| **Vaelix OS Subtitle:** "I made one that makes tradeoffs for me." | "A Linux distribution tailored around my daily workflow." |
| **Vaelix OS Why:** "...starts where I actually work — not where Canonical thinks I should." | "...tailored to my exact daily workflow rather than default Canonical choices." |
| **LOQ Control Why:** "On Windows everything felt polished... on Linux I suddenly lost control." | "Transitioning to Linux meant managing hardware power and thermal layers directly." |
| **LOQ Control v1→v2:** "That's not a UX bug. That's a trust violation. Fixed it at the root." | "This was a critical security design flaw that I resolved at the kernel interface level." |
| **ScholarSync Subtitle:** "...answers with citations... not one PDF at a time." | "...answers queries by synthesizing context across an entire library simultaneously." |
| **ScholarSync What I'd change:** "...batch was right for the hackathon, wrong for production..." | "...opting for batch allowed us to ship a stable build within the 24-hour hackathon window..." |
| **Minor list label:** "// shipped, not padded" | "// completed utilities and smaller builds" |
| **Sovereign Plate sub:** "Real client-facing product, not a demo." | "Deployed client platform managing active table bookings." |
| **The Workbench Subtitle:** "Not a skill list. Only tools I'd defend..." | "Categorized by frequency of use and level of systems integration." |
| **Internals Subtitle:** "Not decorative — real algorithms..." | "Live execution environments for algorithms running client-side." |
| **Build Log Subtitle:** "Not a blog — a record." | "A historical record of technical decisions and architectural revisions." |
| **whoami terminal:** "vaibhav — systems builder, not just a developer" | "vaibhav — systems developer" |
